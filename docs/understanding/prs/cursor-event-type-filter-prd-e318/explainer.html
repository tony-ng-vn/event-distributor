At a glance

Friends can filter the shared feed by event type. Events classify asynchronously after ingest (rules by default; optional InsForge AI). Admins can override and backfill.

Background

Deep background (skippable)

Event Radar stored Luma metadata without categories. Feed filters were only response-state (all/pending/accepted).

What mattered before

This branch adds type columns, a classifier, feed type pills, card labels, admin override, and deploy/migrate notes.

Intuition

Goal: Ship triage by event type without blocking calendar sync or inventing a fragile keyword-only system.

Ingest→async classify→primary_type→feed pills

Insert first so sync stays fast. Untyped rows stay under All; Other means we tried and could not type it. Human overrides stick.

Code walkthrough

Step 1 — Schema + classifier

Migration adds primary_type/type_source. Classifier supports off/mock/rules/insforge with soft-fail.

Step 2 — API + ingest hook

serializeEvent returns type fields; scheduleEventTypeClassification runs after insert; admin PATCH/backfill routes.

Step 3 — Feed UI

Type filter pills compose with pending/accepted; cards show tag-pill labels; admin select overrides.

Step 4 — Harden + deploy

Missing model confidence → fallback; migrate note in production docs; unit tests green.
