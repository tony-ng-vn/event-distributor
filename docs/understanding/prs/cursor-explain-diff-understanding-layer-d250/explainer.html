At a glance

This change adds an understanding layer: literate explainers, embedded quizzes, and optional micro-worlds — one folder per commit and a roll-up per PR — so you can stay in the loop while agents ship code.

Background

Deep background (skip if familiar)

Long agent runs produce large diffs. A raw git diff (the list of line-by-line changes git shows) is hard to retain. Andy Matuschak notes that reading alone often fools us into thinking we understood when we did not.

An understanding layer is teaching material written next to the code: background, intuition, narrative code tour, and a quiz to verify comprehension.

What mattered before this change

The repo already encouraged incremental commits and listed global InsForge skills in AGENTS.md. It lacked a repeatable per-commit teaching workflow and templates tied to git history.

Intuition

Goal: After every agent commit, you get a short lesson about that slice; at PR time you get the full story map.

commit→ understanding-diff.mjs→ explainer.html + quiz→ optional micro-world

Each commit gets its own folder under docs/understanding/commits/; the PR gets a roll-up under docs/understanding/prs/.

Three pieces work together: explain-diff (the essay), quiz (verify you got it), explain-micro-world (learn by doing — only after you pick a proposal).

Code walkthrough

Step 1 — Skills

.cursor/skills/explain-diff/ and explain-micro-world/ define how agents teach: plain language, background before code, quiz rules, micro-world proposals.

Step 2 — Rules

understanding-layer.mdc (always on) plus an extra step in incremental-commits.mdc require explainers after each commit.

Step 3 — Templates + script
npm run understanding:diff -- --commit HEAD
npm run understanding:diff -- --branch my-branch --base main
diff-context.json — machine-readable diff summary the agent reads before writing prose. It does not auto-generate the lesson.

Quiz

1. When should an agent generate a commit-level explainer?

❌ Incremental understanding during long runs is the point.
✅ The understanding-layer rule ties explainers to each commit.
❌ Small commits compound; no file-count gate.
❌ Raw diffs are hard to retain and verify.

2. Correct section order in a literate diff?

❌ Code before intuition assumes missing context.
✅ Catch up, build mental model, then details, then verify.
❌ Teach before testing.
❌ That is raw diff order.

3. What does understanding-diff.mjs produce?

❌ The agent writes the prose.
✅ Gathers git diff context for the agent.
❌ Micro-worlds are separate.
❌ Output is repo-local.

4. When should an agent build a micro-world?

❌ Propose first; human chooses.
✅ Collaboration loop: propose → pick → build.
❌ Optional but first-class.
❌ Also helps APIs, migrations, parsers.

5. Where do commit explainers live?

❌ This repo versions explainers for reviewers.
✅ Durable, commit-linked paths.
❌ Too easy to lose.
